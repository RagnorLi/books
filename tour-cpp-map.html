LAYER 2 · OOP ABSTRACTION
Ch 5–6
类：具体 → 抽象 → 层次
Ch5 三类型：具体类（值语义）→ 抽象类（纯接口）→ 类层次（多态）。Ch6 解决生命周期：构造/析构/拷贝/移动 的「五法则」以及 RAII 资源管理。
// 具体类 (Ch5.2) class Vector { double* elem; int sz; public: Vector(int s) : elem{new double[s]}, sz{s} {} ~Vector() { delete[] elem; } // RAII }; // 抽象类 (Ch5.3) class Container { public: virtual double& operator[](int) = 0; // 纯虚 virtual ~Container() {} }; // 移动语义 (Ch6.2) Vector(Vector&& a); // move ctor Vector& operator=(Vector&& a); // move assign

全书的叙事逻辑

Why a type? → struct / class（Ch2,5）
Why a template? → 重用代码（Ch7）
Why a concept? → 约束模板（Ch8）
Why RAII? → 资源安全（Ch6）

这本书不教什么

这是一本 Tour（观光），不是教程。
不讲每个 API 的细节 → 去 cppreference
不讲最佳实践细节 → 去 C++ Core Guidelines
不讲性能调优 → 去 Herb Sutter 的文章

目标是：在两天内让有经验的程序员看懂现代 C++ 的全貌。
